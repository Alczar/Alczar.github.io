How it works

When one of the players is selected, one of the following IDs, which contains some defined variables, is added to the main container

Most of these variables are used to set the coordinates of the polygon appearing over the hexagonal shape. The coordinates are defined inside the value of a clip-path property and computed by some calc() expressions.

.player[id] .player__ratings::after { clip-path: polygon( /* point 1 */ calc(50% + calc(20% * var(--pac) / 100)) calc(50% - calc(35% * var(--pac) / 100)), /* point 2 */ calc(50% + calc(40% * var(--sho) / 100)) 50%, /* point 3 */ calc(50% + calc(20% * var(--pas) / 100)) calc(50% + calc(35% * var(--pas) / 100)), /* point 4 */ calc(50% - calc(20% * var(--dri) / 100)) calc(50% + calc(35% * var(--dri) / 100)), /* point 5 */ calc(50% - calc(40% * var(--def) / 100)) 50%, /* point 6 */ calc(50% - calc(20% * var(--phy) / 100)) calc(50% - calc(35% * var(--phy) / 100))); }

A simple transition effect over the clip-path is applied with the code below.

transition: clip-path .33s ease-in-out;
